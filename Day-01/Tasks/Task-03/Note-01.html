  1. Add the script in <head>:
    • Placing the script in the <head> loads it before the HTML content is rendered.
    • This can block the rendering of the page until the script is fully loaded and executed. 
    
        <!DOCTYPE html>
        <html>
        <head>
            <title>Script in Head</title>
            <script>
                console.log("Script loaded in <head>");
            </script>
        </head>
        <body>
            <h1>Hello, JavaScript!</h1>
        </body>
        </html>
  2. Add the script at the end of the <body> tag:
    • Placing the script at the end of the <body> ensures the HTML content is loaded first.
    • This improves page load performance as the script doesn't block rendering. 
    
        <!DOCTYPE html>
        <html>
        <head>
            <title>Script at End of Body</title>
        </head>
        <body>
            <h1>Hello, JavaScript!</h1>
            <script>
                console.log("Script loaded at end of <body>");
            </script>
        </body>
        </html>
  3. Use the async attribute:
    • The async attribute loads the script asynchronously without blocking HTML parsing.
    • The script executes as soon as it is downloaded, even if HTML parsing is not complete. 
    
        <!DOCTYPE html>
        <html>
        <head>
            <title>Script with Async</title>
            <script src="script.js" async></script>
        </head>
        <body>
            <h1>Hello, JavaScript!</h1>
        </body>
        </html>
  4. Use the defer attribute:
    • The defer attribute loads the script asynchronously but delays execution until HTML parsing is complete.
    • Scripts with defer execute in the order they appear in the document. 
    
        <!DOCTYPE html>
        <html>
        <head>
            <title>Script with Defer</title>
            <script src="script.js" defer></script>
        </head>
        <body>
            <h1>Hello, JavaScript!</h1>
        </body>
        </html>
  5. Understand the differences:
    • Script in <head>: Blocks HTML parsing until the script is loaded and executed.
    • Script at end of <body>: Ensures HTML is parsed first, improving performance.
    • async: Loads and executes the script asynchronously, without blocking HTML parsing.
    • defer: Loads the script asynchronously but executes it only after HTML parsing is complete.

Steps to Test:

  1. Create an HTML file for each method (e.g., head.html, body-end.html, async.html, defer.html).
  2. Add the corresponding script loading method to each file.
  3. Open each file in a browser and observe the behavior in the console.
  4. Compare the loading and execution times using the browser's DevTools (Network tab).
